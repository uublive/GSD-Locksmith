  <behavior>
    VAL-01 check_phase_gaps():
    - Reads ROADMAP.md content (via git show :.planning/ROADMAP.md in production; via $GSD_TEST_ROADMAP when TESTING_MODE=1)
    - Extracts lines matching: grep -nE '^\- \[.\] \*\*Phase [0-9]+:'
    - Uses awk with match($0, /Phase ([0-9]+):/, arr) to extract sequential numbers and detect gaps
    - For each gap: emits ".planning/ROADMAP.md:LINE: ERROR: Phase gap — expected Phase N, found Phase M" to stderr
    - Emits indented "  Fix: add '- [ ] **Phase N: [Name]** - ...' entry before Phase M at line LINE" to stderr
    - Returns 0 if no gaps, 1 if any gap found (NOT exit — return only)

    VAL-02 check_duplicate_req_ids():
    - Reads REQUIREMENTS.md content (via git show or $GSD_TEST_REQUIREMENTS when TESTING_MODE=1)
    - Extracts REQ-IDs with: grep -oE '\*\*[A-Z]+-[0-9]+\*\*' | sed 's/\*\*//g' | sort | uniq -d
    - For each duplicate ID: finds its line numbers with grep -n, emits one error line per occurrence
    - Error format: ".planning/REQUIREMENTS.md:LINE: ERROR: Duplicate REQ-ID ID"
    - Fix: "  Fix: rename one occurrence of ID to the next available ID in its category"
    - Returns 0 if no duplicates, 1 if any found

    VAL-03 check_state_drift():
    - Reads both STATE.md and ROADMAP.md content (via git show or $GSD_TEST_STATE / $GSD_TEST_ROADMAP when TESTING_MODE=1)
    - Check A — total_phases mismatch: extracts "  total_phases: N" from STATE.md YAML (grep -n 'total_phases:'); counts phases in ROADMAP.md (grep -cE '^\- \[.\] \*\*Phase [0-9]+:' || true); if mismatch, emits error with both values and fix command updating total_phases
    - Check B — active phase invalid: extracts "**Phase:** N" from STATE.md body (grep -n '^\*\*Phase:\*\* '); checks if Phase N exists in ROADMAP.md; if not found, emits error naming the active phase number and a fix command
    - Returns 0 if no drift, 1 if any drift detected

    VAL-04 check_stale_refs():
    - In production: uses git ls-files '.planning/phases' | grep -E '[0-9]+-[0-9]+-PLAN\.md$' to enumerate PLAN.md files; reads each with git show ":PLAN_PATH"
    - In TESTING_MODE=1: reads $GSD_TEST_PLAN_CONTENT for a single plan file at path $GSD_TEST_PLAN_PATH
    - Builds valid REQ-ID set from REQUIREMENTS.md content (same grep pattern as VAL-02)
    - For each PLAN.md: parses frontmatter using awk '/^---/{fm++; next} fm<2 && /^  - [A-Z]+-[0-9]+/{print NR" "$0}' to get line numbers and REQ-IDs between first and second "---" markers
    - For each REQ-ID in requirements: field, checks if it exists in valid_req_ids; if not, emits error naming the stale ID, the plan file path, and the line number
    - Returns 0 if no stale refs, 1 if any found

    Shared behaviors for ALL functions (per D-03, D-04, D-05, D-09, D-10):
    - All error output to stderr (>&2)
    - NEVER use exit — use return only
    - NEVER pipe into a while loop (subshell variable loss per Pitfall 1) — use process substitution < <(...) or temp files
    - All grep -c calls append || true (Pitfall 2 / grep exits 1 on zero count)
    - TESTING_MODE=1 check at top of each function, before any git show call
    - verbose_log() available if sourcing common.sh, but validate.sh can define its own minimal version to avoid requiring jq/gh deps
  </behavior>
  <action>
    Create hooks/lib/validate.sh. Structure:

    1. Header comment block: phase, purpose, decisions implemented
    2. set -euo pipefail
    3. TESTING_MODE check and a small internal _verbose_log() helper (mirrors common.sh verbose_log but without requiring common.sh source)
    4. Four functions in order: check_phase_gaps, check_duplicate_req_ids, check_state_drift, check_stale_refs
    5. Each function starts with local variable declarations (local content errors=0 etc.)
    6. Each function reads content: if TESTING_MODE=1 use the GSD_TEST_* variable, else call git show :.planning/FILE

    Key implementation notes (from RESEARCH.md):
    - VAL-01 gap detection: use the awk-only approach from Pattern 3 "Simpler awk approach" — let awk do sequential state tracking. awk handles prev/curr internally, no subshell issue. Use: grep -nE '^\- \[.\] \*\*Phase [0-9]+:' then pipe to awk using POSIX match: match($0, /Phase [0-9]+/); n = substr($0, RSTART+6, RLENGTH-6)+0. Avoid three-argument match($0, regex, arr) — gawk-only, not available in BSD awk on macOS. Count errors by reading awk output lines into a counter.
    - VAL-02 duplicate detection: two-pass approach. Pass 1: find duplicate IDs with sort|uniq -d. Pass 2: for each dup_id, find line numbers with grep -n. To avoid subshell counter loss: write each error line to a temp file; count temp file lines for return code.
    - VAL-03 state drift: extract total_phases with: echo "$state_content" | grep -n 'total_phases:' | head -1 then parse number. Extract active phase with: echo "$state_content" | grep -n '^\*\*Phase:\*\* ' | head -1. Both are single-line extractions — no subshell counter issue.
    - VAL-04 stale refs: use awk to parse frontmatter in one pass: awk '/^---/{fm++; next} fm==1 && /^  - [A-Z]+-[0-9]+/{print NR" "$0}'. Then check each extracted ID against valid_req_ids (grep -qx). Write errors to temp file to avoid subshell loss; count temp file lines.

    Temp file pattern for functions that need error counting across subshell boundaries:
      local tmpfile; tmpfile=$(mktemp)
      # ... write errors to $tmpfile with: echo "..." >> "$tmpfile"
      errors=$(wc -l < "$tmpfile" | tr -d ' ')
      cat "$tmpfile" >&2
      rm -f "$tmpfile"
      return $errors
