  </read_first>

  <action>
    Create hooks/gsd-status.sh — a read-only standalone script.

    Shebang: `#!/usr/bin/env bash`. Set `set -euo pipefail`.

    Resolve REPO_ROOT: `REPO_ROOT="$(git rev-parse --show-toplevel)"`. Source both libraries in order: `source "$REPO_ROOT/hooks/lib/common.sh"` then `source "$REPO_ROOT/hooks/lib/gist.sh"`.

    Call check_deps and load_config at the top.

    Read the registry: `REGISTRY="$(read_registry)"`. Validate with `echo "$REGISTRY" | jq -e . >/dev/null 2>&1` — exit 2 with error to stderr if not valid JSON.

    Check for empty active claims before rendering: use `echo "$REGISTRY" | jq '[.claims[] | select(.status=="active")] | length'` to get the count. If count is 0, print "No active claims in registry." to stdout and exit 0.

    Render the table header to stdout: print "TYPE     NUMBER  MILESTONE  OWNER       BRANCH                   CLAIMED_AT" followed by a separator line of dashes.

    Render active claims: use jq to filter and format active claims as tab-separated lines, then pipe through `column -t` for alignment. The jq filter: `.claims[] | select(.status=="active") | [.type, (.number|tostring), (.milestone // "-"|tostring), .owner, .branch, .claimed_at] | @tsv`. Pipe the jq output through `column -t -s $'\t'` for aligned columns.

    Sort the output by type then number so milestones appear before phases: add `| sort_by(.type, .number)` to the jq filter before the `@tsv` output. Full filter: `[.claims[] | select(.status=="active")] | sort_by(.type, .number) | .[] | [.type, (.number|tostring), (.milestone // "-"|tostring), .owner, .branch, .claimed_at] | @tsv`.

    All error messages go to stderr. The table output (including header and separator) goes to stdout.

    After creating the file: `chmod 750 hooks/gsd-status.sh`.
  </action>
