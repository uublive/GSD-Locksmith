<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CC runtime → wrapper script stdin | CC pipes JSON payload; content originates from user-typed command — low trust for command_args value |
| wrapper script → claim-number.sh subprocess | MILESTONE_NUM passed as positional arg — validated before passing |
| wrapper script stdout → CC runtime | Must be clean JSON; any non-JSON corrupts CC hook parsing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | cc-claim-phase.sh — command_args parsing | mitigate | Validate MILESTONE_NUM matches ^[1-9][0-9]*$ before passing to subprocess; reject non-integers at entry |
| T-02-02 | Tampering | cc-claim-phase.sh — jq stdin parsing | mitigate | Use jq -r '.command_args // ""' — jq exits non-zero on malformed JSON (set -e traps it); // "" prevents null propagation |
| T-02-03 | Information Disclosure | wrapper scripts stdout | mitigate | All non-JSON output routed to >&2; stdout exclusively jq-formatted JSON; smoke-tested with "pipe to jq ." |
| T-02-04 | Tampering | wrapper scripts — chmod | mitigate | chmod 750 at creation; world-write bit removed |
| T-02-05 | Spoofing | additionalContext content | accept | Content is a plain-text string injected as context, not as executable instruction; no eval path in CC runtime |
| T-02-06 | Denial of Service | /tmp tmpfile accumulation on failure | accept | rm -f in both success and failure paths; residual files are small and benign for 3-person team |
</threat_model>
