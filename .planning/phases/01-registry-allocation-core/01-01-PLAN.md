    hooks/lib/common.sh (per D-11): Bash script with shebang `#!/usr/bin/env bash`. Define three functions: check_deps(), load_config(), verbose_log().

    check_deps() validates three things in order: `command -v jq` (exit 1 with "ERROR: jq is required. Install with: brew install jq"), `command -v gh` (exit 1 with "ERROR: gh CLI is required. Install with: brew install gh && gh auth login"), `gh auth status >/dev/null 2>&1` (exit 1 with "ERROR: gh auth not configured. Run: gh auth login"). All error messages go to stderr (>&2).

    load_config() resolves REPO_ROOT via `git rev-parse --show-toplevel`, reads $REPO_ROOT/.claude/gsd-team.json, extracts GIST_ID via `jq -r '.gist_id'`. Fails with exit 1 on stderr if the config file is missing or if GIST_ID is "null" or empty.

    verbose_log() emits "$*" to stderr prefixed with "[GSD]" only when GSD_VERBOSE is set and non-empty: `[[ -n "$GSD_VERBOSE" ]] && echo "[GSD] $*" >&2`.

    hooks/lib/gist.sh (per REG-03, Pattern 2): Bash script with shebang `#!/usr/bin/env bash`. Set GIST_FILE="registry.json" at the top. Define two functions: read_registry() and write_registry().

    read_registry(): calls `verbose_log "Reading registry from gist $GIST_ID"` then `gh gist view "$GIST_ID" --filename "$GIST_FILE" --raw`. Returns the raw JSON on stdout.

    write_registry(): takes one argument (the JSON content string). Creates a temp file via `mktemp /tmp/gsd-registry-XXXXXX.json`, writes content with `printf '%s' "$1" > "$tmpfile"` (not echo — avoids flag interpretation). Calls `verbose_log "Writing registry to gist $GIST_ID"` then `gh gist edit "$GIST_ID" --filename "$GIST_FILE" "$tmpfile"`. Captures exit code, removes tmpfile with `rm -f "$tmpfile"`, returns the exit code. The tmpfile path argument MUST be last in the `gh gist edit` call — omitting it opens $EDITOR interactively (Pitfall 2).

    After creating the files, run `chmod 750 hooks/lib/common.sh hooks/lib/gist.sh`.

    Create hooks/claim-number.sh — the allocation entry point that Phase 2 CC hooks will call.

    Shebang: `#!/usr/bin/env bash`. Set `set -euo pipefail` at the top.

    Resolve REPO_ROOT: `REPO_ROOT="$(git rev-parse --show-toplevel)"`. Source both libraries: `source "$REPO_ROOT/hooks/lib/common.sh"` then `source "$REPO_ROOT/hooks/lib/gist.sh"`.

    Argument handling: Accept TYPE as $1 (values: "milestone" or "phase"). For "phase" type, also require MILESTONE_NUM as $2. If TYPE is missing or invalid, print usage to stderr and exit 2.

    Gather metadata: owner via `gh api user --jq '.login'`, branch via `git branch --show-current`, claimed_at via `date -u +%Y-%m-%dT%H:%M:%SZ`.

    Read registry: `REGISTRY="$(read_registry)"`. Validate that the result is non-empty valid JSON: `echo "$REGISTRY" | jq -e . >/dev/null 2>&1` — if it fails, print "ERROR: registry is not valid JSON" to stderr and exit 2. This is the Pitfall 4 / security guard for malformed gist content.

    Compute next number: use the jq max+1 pattern with empty-array guard (Pitfall 1 — critical):
    - For milestone: `jq '[.claims[] | select(.type=="milestone" and .status=="active") | .number] | if length == 0 then 1 else max + 1 end'`
    - For phase with MILESTONE_NUM: `jq --argjson m "$MILESTONE_NUM" '[.claims[] | select(.type=="phase" and .status=="active" and .milestone==$m) | .number] | if length == 0 then 1 else max + 1 end'`

    Dry-run guard (per ALLOC-04, D per GSD_DRY_RUN): Before any write, check `[[ -n "$GSD_DRY_RUN" ]]`. If set, print the claim that would be written to stderr (not stdout) with prefix "[DRY RUN]", then exit 0. No gist write occurs.

    Build updated registry JSON: For milestone type (D-08), append TWO entries to .claims in a single jq expression — the milestone claim and phase-1 claim for that milestone. For phase type, append one entry. Each claim object has these exact fields per D-04: type, number, milestone (phase claims only), owner, branch, claimed_at, status ("active"). Pass all dynamic values via jq --arg and --argjson to avoid injection.

    Write and verify (D-09 collision detection): Call `write_registry "$UPDATED_REGISTRY"`. Then immediately re-read: `REGISTRY_AFTER="$(read_registry)"`. Run detect_collision: use `jq --argjson n "$NEXT_NUM" --arg t "$TYPE" --arg o "$owner" '[.claims[] | select(.type==$t and .number==$n and .owner!=$o)] | length > 0'` against REGISTRY_AFTER. If collision returns "true", retry once by recomputing NEXT_NUM from the re-read registry, building a new claim, and writing again. If the second write also collides, print a collision warning to stderr with the competing owner's name and exit 2.

    Success output: On successful write (no collision or resolved collision), print the claimed number(s) to stdout — for milestone: "Claimed milestone $NEXT_NUM and phase 1 of milestone $NEXT_NUM", for phase: "Claimed phase $NEXT_NUM of milestone $MILESTONE_NUM".

    All operational messages (dep errors, collision warnings, verbose logs) go to stderr (>&2). Only the success summary goes to stdout. This is critical for Phase 2 compatibility (Pitfall 5).

    After creating the file: `chmod 750 hooks/claim-number.sh`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| script → GitHub Gist API | Registry JSON returned by gist is untrusted network data; must validate before parsing |
| script → stdin (Phase 2) | When called from CC hooks, JSON arrives on stdin; must validate before interpreting |
| developer → script args | $1 (TYPE) and $2 (MILESTONE_NUM) come from caller; must validate before jq injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | gist.sh read_registry() | mitigate | Validate registry JSON with `jq -e . >/dev/null 2>&1` before any `.claims` access; exit 2 on invalid JSON |
| T-01-02 | Tampering | claim-number.sh arg handling | mitigate | Pass all dynamic values to jq via --arg / --argjson flags, never via string interpolation into filter expression |
| T-01-03 | Information Disclosure | hooks/claim-number.sh stdout | mitigate | All output except success summary routed to stderr (>&2); stdout is clean for Phase 2 JSON contract |
| T-01-04 | Tampering | hooks/ script permissions | mitigate | chmod 750 on all scripts in hooks/; owned by developer user; not world-writable |
| T-01-05 | Spoofing | owner field in claim entry | accept | Owner populated via `gh api user --jq '.login'` which requires authenticated gh session; no additional owner verification needed for 3-person team |
| T-01-06 | Information Disclosure | .claude/gsd-team.json gist_id | accept | Gist ID is team-shared, not secret; gist is not private; no tokens stored in this file; risk is low for internal team tooling |
</threat_model>
