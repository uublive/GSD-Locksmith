  <action>
    Create the .githooks/ directory if it does not exist. Then create .githooks/pre-merge-commit with the following structure:

    1. Shebang: #!/usr/bin/env bash
    2. Header comment: purpose, limitation note (does NOT fire on conflicted merges — Pitfall 5), how to activate (git config core.hooksPath .githooks)
    3. set -euo pipefail
    4. REPO_ROOT="$(git rev-parse --show-toplevel)"
    5. Source validate.sh: source "$REPO_ROOT/hooks/lib/validate.sh"
    6. Branch filter (D-08): CURRENT_BRANCH="$(git rev-parse --abbrev-ref HEAD)"; if neither "development" nor "develop", exit 0 immediately
    7. Error accumulator init: ERRORS=0
    8. Run all four checks using || to capture return code without triggering set -e:
         check_phase_gaps          || ERRORS=$((ERRORS + $?))
         check_duplicate_req_ids   || ERRORS=$((ERRORS + $?))
         check_state_drift         || ERRORS=$((ERRORS + $?))
         check_stale_refs          || ERRORS=$((ERRORS + $?))
    9. If ERRORS > 0: print summary to stderr ".planning: $ERRORS integrity violation(s). Fix above errors and retry the merge." then exit 1
    10. exit 0

    After creating the file: chmod 750 .githooks/pre-merge-commit (per Phase 2 convention of chmod 750 for hook wrappers).

    IMPORTANT: The four check functions must NOT use set +e / set -e sandwich because the functions use return, not exit. The || ERRORS=$((ERRORS + $?)) pattern safely captures the function return code without triggering set -e. This is different from the subprocess pattern in 02-01 which needed set +e for subprocess exit codes.

    IMPORTANT: Do NOT add any code that reads or modifies staging area, git config, or index. This is a read-only validation hook. No writes to the repo occur in this hook.
  </action>

|----------|-------------|
| git merge command → pre-merge-commit | Hook executes in the developer's shell with their credentials; reads index via git show |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | pre-merge-commit — source path injection via REPO_ROOT | mitigate | REPO_ROOT set by `git rev-parse --show-toplevel` (git-controlled, not user input); source path is fixed relative to repo root |
| T-03-06 | Tampering | Hook bypass via `git merge --no-verify` | accept | Small trusted team (3 devs); documented limitation in hook header; accepted per project scope |
| T-03-07 | Denial of Service | Malformed planning file causing awk/grep to hang | accept | Planning files are team-authored markdown; no external input; infinite loop risk is negligible |
| T-03-08 | Repudiation | Hook blocking merge with no audit trail | accept | stderr output is visible in terminal; developer sees all errors; no silent failures |
