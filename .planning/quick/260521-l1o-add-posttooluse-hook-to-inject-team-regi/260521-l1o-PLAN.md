# Plan: Registry Ownership Context Hook

## Objective

When Claude runs `gsd-sdk query roadmap.analyze` or `gsd-sdk query init.progress` (via Bash), inject an ownership summary from the team registry as `additionalContext`. This tells Claude who owns which milestones/phases so it makes better routing suggestions without hiding information.

## Tasks

### Task 1: Create `.gsd/ownership-context.sh`

PostToolUse hook script for Bash tool calls.

**Logic:**
1. Read stdin JSON, extract `tool_input.command`
2. Fast-path exit if command doesn't contain `gsd-sdk query roadmap` or `gsd-sdk query init`
3. Read team registry from orphan branch (reuse `common.sh` + `registry.sh`)
4. Get current user via `gh api user --jq '.login'`
5. Filter to active claims only
6. Build ownership summary grouped by milestone
7. Mark current user's claims with `← you`
8. Output JSON with `additionalContext` containing the summary

**Example output injected as additionalContext:**
```
## Team Registry: Ownership Context
- Milestone 1.11: alex (branch: feat/payments)
- Milestone 1.13, phases 37-42: alex (branch: feat/search)
- Milestone 1.15, phases 52-56: matteo (branch: feat/dashboard) ← you
```

**Edge cases:**
- Empty registry or no active claims → exit 0 silently (no context to add)
- Registry unreachable → exit 0 silently (don't block GSD workflows)
- Only own claims → still show them (so user sees "all items are yours")

**Files:** `.gsd/ownership-context.sh`
**Verify:** `echo '{"tool_name":"Bash","tool_input":{"command":"gsd-sdk query roadmap.analyze"}}' | bash .gsd/ownership-context.sh` outputs valid JSON with additionalContext
**Done:** Script exists, is executable, handles fast-path and error cases

### Task 2: Wire hook into `.claude/settings.json`

Add a PostToolUse entry for Bash matcher pointing to the ownership-context script.

**Change:** Add to the existing `hooks` object:
```json
"PostToolUse": [
  {
    "matcher": "Bash",
    "hooks": [
      {
        "type": "command",
        "command": "${CLAUDE_PROJECT_DIR}/.gsd/ownership-context.sh"
      }
    ]
  }
]
```

**Also update:** `install-hooks.sh` to include the new PostToolUse hook entry so new clones get it automatically.

**Files:** `.claude/settings.json`, `.gsd/install-hooks.sh`
**Verify:** `jq '.hooks.PostToolUse' .claude/settings.json` shows the new entry
**Done:** settings.json has PostToolUse entry, install script updated
